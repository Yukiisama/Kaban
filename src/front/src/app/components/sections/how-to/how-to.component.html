Documentation du site
Projects
Cette page regroupe les différents projets existants. Chaque projet est représenté par son nom et sa description. On peut voir « l’avancé » d’un projet (pourcentage de tasks réalisées) grâce à une barre de progression à droite de chaque projets.

Créer un nouveau projet :

- Cliquer sur le bouton « Nouveau projet » pour afficher le formulaire.

- Remplir les champs « Nom » et « Description » du formulaire (attention : deux projets ne peuvent avoir un même nom )

- Cliquer sur le bouton « Enregistrer » pour fermer le formulaire et ajouter le nouveau projet.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau projet.

Éditer un projet existant :

- Cliquer sur le bouton « Éditer » du projet correspondant pour afficher le formulaire pré-rempli de modification.

- Réaliser les modifications voulues (attention : le changement du nom du projet pour un nom déjà utilisé entraînera la non-validation des modifications)

- Cliquer sur le bouton « Enregistre » pour fermer le formulaire et appliquer les modifications réalisées.

- Cliquer sur la croix en haut à droite du formulaire ou sur une zone à l’extérieur du formulaire entraînera la fermeture de celui-ci sans l’ajout du nouveau projet.

Supprimer un projet existant :

-Cliquer sur le bouton « Supprimer » du projet correspondant (attention : l’action est immédiate et irréversible)

Rechercher un projet existant :

- Entrer le(s) mot(s) clé(s) dans la barre de recherche.

- Cliquer sur le bouton « Rechercher » pour mettre à jour la liste des projets.

- Cliquer sur le bouton « Annuler la recherche » remettra la liste à son état initiale. (il n’est pas nécessaire de cliquer sur ce bouton pour réaliser une nouvelle recherche il suffit de changer la recherche dans la barre de recherche et cliquer sur le bouton « Rechercher »)

Accéder à un projet :

- Cliquer sur le titre du projet en question ouvrira le projet

- Cliquer sur le dernier onglet du menu à droite (onglet projets) permet de revenir en arrière.

Backlog

La section Backlog du projet permet de créer, de modifier et d'organiser des user stories dans des sprints.

Il est donc possible de créer des sprints, puis d'en démarrer un. Il sera accessible dans la section Active Sprint

Tasks

La section Tasks du projet permet de créer, de modifier et de gérer les tâches du projet. Les tâches peuvent se trouver dans 3 colonnes :

TODO : Personne ne travaille sur cette tâche.

DOING : La tâche est spécifiée et un développeur s'en charge.

DONE : Toutes les DOD de la tâche sont satisfaites.

Une tâche doit contenir des Definiton of Done avant de pouvoir être passée en DOING, et celles-ci doivent être satisfaites pour le passer en DONE.

Une tâche peut être dépendante de plusieurs autres tâches. Une tâche ne peut pas être dépendante d'une tâche qui est déjà dépendante de celle-ci.
